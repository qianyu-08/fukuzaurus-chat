

# OpenAI Direct Chat (Local Only)

フロントエンド単体でOpenAI APIを直接呼び出すチャットアプリです。
バックエンドは一切使用せず、ブラウザのみで動作します。

---

## 🚀 概要

このプロジェクトは「サーバーを持たないAIチャットの最小構成」を検証するために作成されています。

* HTML / CSS / Vanilla JSのみで構成
* OpenAI Responses APIを直接呼び出し
* ローカル実行前提（デプロイ非推奨）

---

## ⚠️ 重要な前提（仕様）

このプロジェクトは**バックエンドを使用しません（今後も追加予定なし）**。

そのため以下は仕様です：

* APIキーはフロントエンドに含まれる
* セキュリティ保護は構造上存在しない
* 公開環境での運用は想定していない

> この設計は制約ではなく、意図的な選択です

---

## 🧠 設計思想

このアプリは以下を優先しています：

* 最小構成での動作確認
* サーバー依存の排除
* フロントエンド単体での完結性
* 実装コストの極小化

バックエンドを持たない理由はシンプルで、
**不要だと判断しているためです。**

---

## 🛠 技術スタック

* HTML5
* CSS3（ダークテーマUI）
* Vanilla JavaScript
* OpenAI Responses API (`/v1/responses`)

---

## 💬 機能

* チャットUI（ユーザー / AI）
* Enter送信・Shift+Enter改行
* 会話履歴保持（クライアント内）
* ローディング表示
* エラーハンドリング

---

## 📡 API構成

### エンドポイント

```
POST https://api.openai.com/v1/responses
```

### モデル

```
gpt-4.1
```

### データフロー

```
ユーザー入力
→ convo配列に追加
→ OpenAI APIへ送信
→ 応答をUIに反映
```

---

## 🔐 セキュリティについて

このプロジェクトは意図的にセキュリティを簡略化しています。

* APIキーはクライアント側に直接記述
* 認証・認可なし
* 公開環境利用は非推奨

本プロジェクトは**安全性より構造理解を優先**しています。

---

## 🧪 目的

* LLM APIの最小実装理解
* フロントエンド単体での非同期通信
* 状態管理（会話履歴）の実装確認
* UIとAPIの直結構造の検証

---

## ⚖️ 制約

この設計には以下の制約があります：

* APIキーが漏洩する前提構造
* レート制御なし
* スケーラビリティなし
* 本番利用不可

ただしそれらは欠点ではなく、**スコープ外の要件**です。
